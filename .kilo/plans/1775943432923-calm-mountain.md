# Pricing Page CSS Fix Plan

## Scope
Only **active** components (not hidden by `{false && ...}` in pricing-section.tsx):
- `pricing-section.tsx` — main wrapper
- `credit-pack-prices.tsx` — credit pack cards
- `general-features-section.tsx` — features grid
- `pricing-description.tsx` — description wrapper
- `credit-usage.tsx` — credit usage table
- `transcription-usage.tsx` — transcription table
- `which-models.tsx` — model recommendations
- `currency-tabs.tsx` — currency selector

**Excluded** (inactive): `pricing-cards.tsx`, `feature-comparison-table.tsx`

## Issues & Fixes

### 1. Hardcoded blue colors → semantic design tokens
Per AGENTS.md: `bg-primary text-primary-foreground` not `bg-blue-500 text-white`, `text-primary` not `text-blue-500`, `border-primary` not `border-blue-500`.

| File | Current | Fix |
|------|---------|-----|
| credit-pack-prices.tsx | `border-t-blue-500`, `hover:border-t-blue-400 dark:hover:border-t-blue-600` | `border-t-primary`, `hover:border-t-primary/80` |
| credit-pack-prices.tsx | `bg-blue-500 hover:bg-blue-600 text-white` (Purchase button) | `bg-primary text-primary-foreground hover:bg-primary/90` |
| credit-pack-prices.tsx | `bg-transparent hover:bg-blue-50 dark:hover:bg-blue-900/20 border border-blue-500 text-blue-600 dark:text-blue-400 hover:text-blue-700 dark:hover:text-blue-300` (outline buttons) | `variant="outline" className="border-primary text-primary hover:bg-primary/10 hover:text-primary"` |
| general-features-section.tsx | `text-blue-500` on Check icons | `text-primary` |
| credit-usage.tsx | `text-blue-500` on Check icons | `text-primary` |
| pricing-description.tsx | `text-blue-500 hover:text-blue-600` on links | `text-primary hover:text-primary/80` |

### 2. Inconsistent Card padding
Card components provide `px-4` by default on CardContent. Some cards override with `px-8` or `px-6`, creating visual inconsistency.

| File | Current | Fix |
|------|---------|-----|
| pricing-description.tsx (3 cards) | `px-8` on CardContent | Remove `px-8`, use default `px-4` with `space-y-4` |
| which-models.tsx | `px-8` on CardContent | Remove `px-8`, use default `px-4` with `space-y-4` |
| transcription-usage.tsx | `px-8` on CardContent | Remove `px-8`, use default `px-4` with `space-y-4` |
| general-features-section.tsx | `px-6` on CardContent | Remove `px-6`, use default `px-4` with `space-y-4` |
| credit-pack-prices.tsx (outer card) | CardContent with `space-y-4` | Already correct |
| credit-pack-prices.tsx (inner cards) | Uses CardHeader/CardContent/CardFooter | OK, uses default padding |

### 3. Nested card rounded override
Inner credit pack cards use `rounded-md` which overrides Card's default `rounded-xl`.

| File | Current | Fix |
|------|---------|-----|
| credit-pack-prices.tsx inner cards | `rounded-md` | Remove, let Card's default `rounded-xl` apply |

### 4. Inconsistent section widths
The outer section uses `max-w-6xl` but all inner cards use `max-w-5xl`. This creates wasted space.

| File | Current | Fix |
|------|---------|-----|
| pricing-section.tsx | `max-w-6xl` | `max-w-5xl` to match inner cards |
| pricing-description.tsx | `max-w-6xl` | `max-w-5xl` to match inner cards |

Since inner cards also have `max-w-5xl mx-auto`, they're already constrained — the section wrapper doesn't need to be wider. Aligning to `max-w-5xl` removes the gap, and we can remove `mx-auto` from inner cards since the parent constrains width.

### 5. Duplicate `id="pricing"` 
Both `pricing-section.tsx` and `pricing-description.tsx` use `id="pricing"`. Remove from one (keep on pricing-section since it's the first section).

### 6. Minor inconsistencies
- `pricing-description.tsx`: Section has `py-4` vs `pricing-section.tsx`'s `py-16`. Change to `py-12` for visual consistency.
- `credit-usage.tsx`: `text-green-500` for discount percentage — keep as-is since green = savings is intentional.
- `credit-usage.tsx`: `text-primary font-medium` for discounted price is correct already.
