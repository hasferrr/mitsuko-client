# Multiselect for Project Inner Items (Translations/Transcriptions/Extractions)

## Goal

Add multiselect to the project main page (`ProjectMain`) so users can select multiple translations, transcriptions, and/or extractions, then bulk Delete or Move them.

---

## Design Decisions (from interview)

| Decision            | Choice                                                                                       |
| ------------------- | -------------------------------------------------------------------------------------------- |
| Scope               | Per-tab — select mode applies to current tab's items only                                    |
| Overview sections   | Linked/shared — one select mode spans all 3 sections in Overview                             |
| Entry point         | Toggle button in each section header (next to Settings + New)                                |
| Toggle deactivation | Any section toggle enters/exits for the whole tab                                            |
| Actions             | Delete + Move (no Export, no Archive)                                                        |
| Mixed-type Move     | All selected items go to same target project via one dialogue                                |
| Confirmations       | Delete requires confirmation (count by type), Move doesn't (dialogue itself is confirmation) |
| Checkbox placement  | Replaces grip handle — same position, no layout shift                                        |
| Post-action         | Clear selection and exit select mode                                                         |
| Exit methods        | Section toggle, Cancel on floating bar, Escape key                                           |
| Select All          | Split button with per-type dropdown in Overview tab; simple toggle in single-type tabs       |
| Bar component       | New dedicated `ProjectItemSelectionBar`                                                      |
| Selection hook      | New `useProjectItemSelection` hook in `ProjectMain`                                          |
| Active items        | Selectable but Delete/Move disabled when any selected item has active operations             |
| Tab switch          | Exits select mode and clears selection                                                       |
| Execution           | Sequential delete/move                                                                       |
| Drag-reorder        | Disabled in select mode                                                                      |
| Move dialogue       | New `BulkMoveDialogue` with item count breakdown                                             |
| Delete dialog       | Count by type ("Delete 2 translations, 1 transcription?")                                    |

## Implementation Steps

### Step 1: Create `useProjectItemSelection` hook

**New file**: `src/hooks/project/use-project-item-selection.ts`

```ts
type ItemType = "translation" | "transcription" | "extraction"

interface UseProjectItemSelectionProps {
  translations: Translation[]
  transcriptions: Transcription[]
  extractions: Extraction[]
  currentProjectId: string
  currentTab: string  // "overview" | "translations" | "transcriptions" | "context-extractor"
}
```

State:
- `isSelecting: boolean`
- `selectedIds: Map<string, ItemType>` — maps each selected item ID to its type
- `isDeleteDialogOpen`, `isMoveDialogOpen`, `isProcessing`

Computed (useMemo):
- `selectedCounts: { translations: number, transcriptions: number, extractions: number }`
- `allIds: string[]` — all item IDs visible in current tab
- `allSelected: boolean`
- `hasActiveOperations: boolean` — checks if any selected ID is in the corresponding service store's active set (`isTranslatingSet`, `isTranscribingSet`, `isExtractingSet`)

Actions:
- `toggleSelectMode()` — flip `isSelecting`, clear selection on exit
- `exitSelectMode()` — set `isSelecting = false`, clear selection
- `handleSelectToggle(id, type)` — add/remove from Map
- `handleSelectAllToggle()` — if all selected, clear; else select all visible IDs
- `handleSelectTypeOnly(type)` — select only IDs of that type (for Overview dropdown)
- `handleDeleteSelected()` — sequential: for each selected item, call `deleteTranslationDb`/`deleteTranscriptionDb`/`deleteExtractionDb` + `updateProjectItems`, then update local state via `setTranslations`/`setTranscriptions`/`setExtractions`, exit select mode
- `handleMoveSelected(targetProjectId)` — sequential: for each selected item, run DB transaction (remove from source project, add to target, update item's `projectId`), update stores, update local state, exit select mode

The hook receives `setTranslations`, `setTranscriptions`, `setExtractions` as params (or callbacks) to update local state after bulk operations.

### Step 2: Create `ProjectItemSelectionBar` component

**New file**: `src/components/shared/project-item-selection-bar.tsx`

Visual pattern follows `CardGridSelectionBar`:
- Fixed bottom bar with slide-up animation
- Selected count text ("3 selected")
- **Delete** button (destructive variant, disabled when `hasActiveOperations`)
- **Move** button (outline variant, disabled when `hasActiveOperations`)
- **Select All / Deselect All** — split button:
  - In Overview tab when multiple types exist: main click toggles all, dropdown arrow offers "Select Translations Only", "Select Transcriptions Only", "Select Extractions Only"
  - In single-type tab or when only one type visible: simple toggle
  - When all selected: "Deselect All" (single click, no dropdown)
- **Cancel** button (exits select mode)

Props:
```ts
interface ProjectItemSelectionBarProps {
  open: boolean
  selectedCount: number
  selectedCounts: { translations: number; transcriptions: number; extractions: number }
  isProcessing: boolean
  allSelected: boolean
  hasActiveOperations: boolean
  currentTab: string
  hasTranslations: boolean
  hasTranscriptions: boolean
  hasExtractions: boolean
  onDelete: () => void
  onMove: () => void
  onSelectAllToggle: () => void
  onSelectTypeOnly: (type: ItemType) => void
  onCancel: () => void
}
```

### Step 4: Extend `DeleteDialogue` for bulk counts (or create `ItemDeleteDialogue`)

**Option A** (simpler): Extend existing `DeleteDialogue` to accept optional `message?: string` prop. When provided, use it instead of the default "Are you sure you want to delete this item?".

**Option B**: Create `ItemDeleteDialogue` — same as DeleteDialogue but shows type-based count: "Are you sure you want to delete 2 translations and 1 transcription?"

**Recommendation**: Option A — add `message?: string` prop to existing `DeleteDialogue`. The caller formats the message string with the count breakdown.

### Step 5: Modify `ProjectItemList`

**File**: `src/components/project/project-item-list.tsx`

Add props:
```ts
selectMode?: boolean
selected?: boolean
onSelectToggle?: () => void
```

Changes:
- When `selectMode`:
  - Replace grip handle (`<button {...attributes} {...listeners}>` + `<GripVertical>`) with `<Checkbox checked={selected} onCheckedChange={onSelectToggle} />`
  - Hide action buttons (Move, Edit, Delete)
  - Card click toggles selection (instead of navigating)
  - Apply selection highlight: `ring-primary bg-primary/5 dark:bg-primary/10` when `selected`
  - Disable `useSortable` drag: pass `disabled: true` to `useSortable({ id, disabled: selectMode })`
- When not `selectMode`: current behavior unchanged

Reference pattern: `SortableBatchFile` already does this with `selectMode`, `selected`, `onSelectToggle` props.

### Step 7: Modify `ProjectMain`

**File**: `src/components/project/project-main.tsx`

Changes:
- Track current tab value via `Tabs` `onValueChange` → `currentTab` state
- When `currentTab` changes, call `exitSelectMode()` from the hook
- Initialize `useProjectItemSelection` hook with translations, transcriptions, extractions, currentProjectId, currentTab
- Pass select mode props down to each list component (all 4 tab content areas)
- Add Escape key listener (`useEffect` with `keydown` handler) — calls `exitSelectMode()`
- Render `ProjectItemSelectionBar` at bottom of the component div (always present, controls its own visibility via `open` prop)
- Render `BulkMoveDialogue` (controlled by `isMoveDialogOpen` from hook)
- Render `DeleteDialogue` (controlled by `isDeleteDialogOpen` from hook, with custom message)
- The hook needs `setTranslations`, `setTranscriptions`, `setExtractions` as params to update local state after bulk operations

## Edge Cases

- **Active operations**: Items being translated/transcribed/extracted can be selected, but Delete and Move are disabled (via `hasActiveOperations`)
- **Tab switch**: Exits select mode, clears selections
- **Delete all items in a section**: Section shows empty state inside select mode, select mode stays active
- **Move mid-operation items**: Blocked by `hasActiveOperations` check
- **Overview + single-type tab share same items**: Selection state is per-tab (cleared on tab switch), so no stale state issues
- **Duplicate list renders**: ProjectMain renders the same list component twice (Overview tab + single-type tab). Only the active tab's instance is mounted by Tabs, so props flow correctly.
