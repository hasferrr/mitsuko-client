# Multiselect Feature for Project & Batch Card Grids

## Scope

Add multiselect to the **card grid level** of the project page and batch page (the "Select a Project" / "Select a Batch" views). Inner item lists (translations/transcriptions/extractions inside a selected project) are **skipped for now**.

---

## Design Decisions

### Actions on Selected Cards
- **Delete** — with confirmation dialog
- **Archive** — separate button for active selections, with confirmation dialog
- **Unarchive** — separate button for archived selections, with confirmation dialog
- **Export** — single JSON file using existing `DatabaseExport` schema, containing all selected projects/batches and their related entities

### Entry/Exit Select Mode
- **Enter**: Toggle button ("Select") in header area next to "Create New Project/Batch"
- **Exit**: Header toggle button ("Cancel"), "Cancel" button on floating bar, Escape key
- Select toggle **disabled** when no cards exist at all (active + archived)

### Selection Behavior
- **Single select mode spanning both sections** (active + archived)
- Archived section **auto-expands** when entering select mode; **restores previous collapsed state** when exiting
- **Drag-reorder disabled** in select mode (checkboxes replace grip handles)
- **Checkbox** appears at top-left corner of card (overlaying card header)
- Selected cards: `ring-primary` + `bg-primary/5 dark:bg-primary/10` tint
- **Select All / Deselect All** selects/deselects **all cards** (active + archived)

### Floating Action Bar
- **Position**: Fixed bottom of viewport
- **Visibility**: Appears only when items are selected (hidden at 0 selections and when not in select mode)
- **Contents**:
  - Selected count text ("3 selected")
  - **Delete** (destructive variant)
  - **Archive** (outline, only shown when active items are selected)
  - **Unarchive** (outline, only shown when archived items are selected)
  - **Export** (outline)
  - **Select All / Deselect All** — split button: main click toggles all, dropdown arrow with options "Select Active Only" / "Select Archived Only". When all selected, main label shows "Deselect All" (one click clears everything, no dropdown).
  - **Cancel** button (exits select mode)
- **Animation**: Subtle slide-up entrance

### After Bulk Action
- **Exit select mode entirely** after any bulk action (delete, export, archive/unarchive). The user completed their task — they can re-enter select mode if needed.

### Implementation Strategy
- **Single combined hook** `useCardGridSelection` — manages both active and archived selections together; reusable by both Project and Batch pages
- **Shared component** `CardGridSelectionBar` — floating action bar
- **New db-io function** `exportProjects(projectIds: string[])` — merges multiple projects into single DatabaseExport JSON
- **Sequential** deletion/archiving (simpler, easier error handling)
- **Independent** select mode state per page (project vs batch)

## Edge Cases

- Deleting/archiving the currently selected `currentProject` — should also clear `currentProject` in store
- Export with 0 selected — action buttons disabled
- Rapid select/deselect — Set-based state handles this naturally
- Archived section was manually expanded before entering select mode — should not collapse it on exit (preserve user intent)
- Empty active section but has archived — Select button still enabled, auto-expands archived on enter
