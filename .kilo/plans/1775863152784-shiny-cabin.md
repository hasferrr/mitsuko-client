# Fix textarea overflow for long unbroken text

## Problem
Two textareas overflow horizontally when containing long text without spaces:
- `src/components/library/create-edit-instruction-dialog.tsx:110`
- `src/components/library/public-library.tsx:396`

## Root Cause
Both textareas are missing `break-words` CSS (`overflow-wrap: break-word`), which forces long unbroken strings to wrap at the container boundary.

## Fix
Add `break-words` to the `className` of both Textarea components:

1. **`create-edit-instruction-dialog.tsx` line 110**: Change  
   `"min-h-[100px] max-h-[300px] overflow-y-auto resize-none"` →  
   `"min-h-[100px] max-h-[300px] overflow-y-auto resize-none break-words"`

2. **`public-library.tsx` line 396**: Change  
   `"min-h-[100px] max-h-[300px] overflow-y-auto resize-none"` →  
   `"min-h-[100px] max-h-[300px] overflow-y-auto resize-none break-words"`
