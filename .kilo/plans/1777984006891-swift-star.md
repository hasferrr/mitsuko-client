# Archive Confirmation Dialog + Deselection

## Context

The `Project` and `Batch` list views already support archiving via card dropdowns (immediate, no confirmation). `project-header.tsx` has an Archive button in the detail view that also fires immediately. The requirement is:

1. **Confirmation dialog** when the Archive button in the detail-view header is clicked.
2. **Deselect the project** when archiving from a detail view so the user returns to the list.

## Decisions (already resolved with user)

- Reusable `ArchiveDialog` component (not inlined).
- Dialog appears **only for detail-view header buttons** (`project-header.tsx`, `batch-main.tsx`). List-view card dropdowns archive immediately (no dialog).
- Dialog only for **Archive**, not Unarchive.
- `hasActiveOperations` remains a **pre-check** that toasts and blocks the dialog from opening.
- `batch-main.tsx` currently has **no** archive button in its header; one will be added for parity with `project-header.tsx`.

## Plan

### 1. Create `src/components/ui-custom/archive-dialog.tsx`

Reusable confirmation dialog using the project's existing `Dialog` primitive (same pattern as `DeleteDialogue` and the Convert dialog in `project-header.tsx`).

**Props:**
```ts
interface ArchiveDialogProps {
  isOpen: boolean
  onOpenChange: (open: boolean) => void
  onConfirm: () => void
  isProcessing?: boolean
  title?: string        // "Archive Project" or "Archive Batch"
  description?: string  // default: "Are you sure you want to archive this project? It will be moved to the Archived section."
}
```

- `Dialog` with `DialogHeader` (title), body text, `DialogFooter` with Cancel + Archive buttons.
- Archive button uses `variant="outline"` (not destructive), disabled while `isProcessing`.
- No logic inside the component — purely presentational.

### 2. Update `src/components/project/project.tsx` (list view)

- No dialog here. `SortableProjectCard` dropdown Archive item keeps current behavior: calls `handleToggleArchive` immediately.
- `ArchivedProjectCard` dropdown Unarchive item: unchanged (no dialog).

### 3. Update `src/components/batch/batch.tsx` (list view)

- No dialog here. `SortableBatchCard` dropdown Archive item keeps current behavior: calls `handleToggleArchive` immediately.
- `ArchivedBatchCard` dropdown Unarchive item: unchanged.

### 4. Update `src/components/project/project-header.tsx` (detail view)

- Add state: `isArchiveDialogOpen`.
- Archive button click handler: check `hasActiveOperations(currentProject.id)` first. If blocked, toast and return. Otherwise open dialog.
- `handleConfirmArchive`:
  1. `await updateProjectStore(currentProject.id, { isArchived: true })`
  2. If success: toast "Project archived"
  3. Deselect: `setCurrentProject(null)`
  4. If `currentProject.isBatch`: `router.push("/batch")` (batch projects viewed via "See as Project" are on `/project` and must navigate back to `/batch`)
  5. Close dialog

- `hasActiveOperations` must be available here. Currently `project-header.tsx` does **not** import the service stores. Add the same `hasActiveOperations` helper used in `project.tsx` (import `useTranslationStore`, `useTranscriptionStore`, `useExtractionStore`, read `.getState()`).

### 5. Update `src/components/batch/batch-main.tsx` (detail view)

This file currently has **no** archive button in its header. Add one for parity.

- Add imports: `Archive` icon, `ArchiveDialog`, service stores for `hasActiveOperations`.
- Add state: `isArchiveDialogOpen`.
- Add archive button next to the existing delete button in the header.
- Click handler: `hasActiveOperations` pre-check → toast or open dialog.
- `handleConfirmArchive`:
  1. `await updateProject(currentProject.id, { isArchived: true })`
  2. If success: toast "Batch archived"
  3. Deselect: `setCurrentProject(null)` (parent `batch.tsx` will show the batch list)
  4. Close dialog

### 6. Type-check and lint

Run `bun typecheck` and `bun lint` to verify.

- `updateProject` already updates `currentProject` in the store if it matches the updated ID, so `setCurrentProject(null)` is the only extra step needed for deselection.
- Toast messages stay consistent with existing ones: "Project archived", "Batch archived", "Failed to archive project".
