# Plan: Archive Support for Batch Projects

## Summary

Extend the existing project archiving system to batch projects, mirroring the non-batch project page's active/archived split layout on the batch page.

## Decisions

- Batch page mirrors project page layout: active sortable grid + archived section below with border-t separator and count badge
- Both active and archived batch cards get a MoreHorizontal dropdown with **only** Archive (active) or Unarchive (archived)
- No Export or Delete in batch card dropdowns — those remain accessible from the project header
- Separate `ArchivedBatchCard` component (opacity-60, no drag handle, dropdown with Unarchive only)
- Same `hasActiveOperations()` guard blocks archiving if items are running
- Empty state shows only when both active AND archived batch lists are empty
- Skeleton count references `activeBatchProjects.length` (fallback 3)
- Archived batch projects excluded from SortableContext (not draggable)
- Drag reorder uses full `projects` array for findIndex/arrayMove (same as project page)
- Sidebar already handles archived batches correctly (hidden unless currentProject)
- Dashboard already filters archived correctly
- Project page remains unchanged (non-batch only)
- `handleToggleArchive` and `hasActiveOperations` defined locally in batch.tsx (same pattern as project.tsx)
- No DB/type changes needed — `isArchived` already exists on Project, migration v25 already backfills it

## Files Changed

### `src/components/batch/batch.tsx` (~+80 lines)

1. Split `batchProjects` into:
   ```ts
   const activeBatchProjects = batchProjects.filter(p => !p.isArchived)
   const archivedBatchProjects = batchProjects.filter(p => p.isArchived)
   ```
2. Add `hasActiveOperations(projectId)` — same logic as project.tsx (reads translation/transcription/extraction service stores)
3. Add `handleToggleArchive(projectId, archive)` — same pattern: guard with `hasActiveOperations`, await `updateProject`, toast success/error
4. Import service stores and icons: `Archive`, `ArchiveRestore`, `MoreHorizontal` from lucide-react; `DropdownMenu*` from ui; `useTranslationStore`, `useTranscriptionStore`, `useExtractionStore`
5. Update SortableContext items to `activeBatchProjects.map(p => p.id)`
6. Render `SortableBatchCard` only for active projects
7. Add archived section below the DndContext (same structure as project.tsx):
   - border-t separator, Archive icon + "Archived" label + count badge
   - Grid of `ArchivedBatchCard` components
8. Update empty state condition to `activeBatchProjects.length === 0 && archivedBatchProjects.length === 0`
9. Update skeleton count to `activeBatchProjects.length > 0 ? activeBatchProjects.length : 3`
10. Update `handleDragEnd` to use `batchProjects` (full batch list including archived) for findIndex/arrayMove — same as project page uses full `projects` array

### `src/components/batch/sortable-batch-card.tsx` (~+25 lines)

1. Add `DropdownMenu`, `DropdownMenuContent`, `DropdownMenuItem`, `DropdownMenuTrigger` imports
2. Add `Archive` icon import
3. Add `onToggleArchive: (id: string, archive: boolean) => void` prop
4. Insert MoreHorizontal dropdown trigger between title and GripVertical
5. Dropdown contains single item: Archive (with Archive icon)
6. Stop click propagation on dropdown trigger and menu item

### `src/components/batch/archived-batch-card.tsx` (new file, ~50 lines)

1. Copy structure from `ArchivedProjectCard` in project.tsx, adapted for batch
2. Card with `opacity-60`, no drag handle, no sortable
3. Header: CardTitle + MoreHorizontal dropdown (single item: Unarchive with ArchiveRestore icon)
4. Props: `project: Project`, `onToggleArchive: (id: string, archive: boolean) => void`
5. Content: same item count + created date as SortableBatchCard
6. Click navigates into project (via `onSelect` prop)
