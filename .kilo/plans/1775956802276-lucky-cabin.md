## Cat 4: Button size variants instead of manual overrides

### `project-item-list.tsx` lines 207–233 — 3 ghost icon buttons

Three buttons with `size="sm" className="h-6 min-w-4 md:min-w-6 p-0"` each contain only a single icon. The `size="sm"` gives h-7 but `h-6` overrides it to h-6, and `p-0` removes default padding.

**Fix:** Switch to `size="icon-xs"` (provides `size-6` = h-6 w-6) and remove `h-6 p-0 min-w-4 md:min-w-6`:
```tsx
// Before (×3)
<Button variant="ghost" size="sm" className="h-6 min-w-4 md:min-w-6 p-0" ...>

// After
<Button variant="ghost" size="icon-xs" ...>
```

## Cat 10: Misc redundant classes

### `project-main.tsx` line 129 — TabsList `flex` redundant

TabsList provides `inline-flex` (which includes flex layout). Adding `flex` overrides `inline-flex` to `display: flex`. Since `flex-wrap` works with both `inline-flex` and `flex`, and the TabsList is inside a container where block vs inline doesn't meaningfully differ, the `flex` override is unnecessary.

**Fix:**
```tsx
// Before
<TabsList className="bg-card border border-border p-1 h-fit flex flex-wrap">

// After
<TabsList className="bg-card border border-border p-1 h-fit flex-wrap">
```
