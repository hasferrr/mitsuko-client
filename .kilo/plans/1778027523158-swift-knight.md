# Refactor: Extract Shared Project Actions Hook

## Context
The batch page (`batch.tsx`) card dropdowns currently only show **Archive/Unarchive**, while the project page (`project.tsx`) shows **Export**, **Archive**, and **Delete**. The export, delete, archive, and active-operations-check logic is duplicated across four locations:
- `src/components/project/project.tsx` (project list)
- `src/components/batch/batch.tsx` (batch list)
- `src/components/sidebar/app-sidebar-projects.tsx` (sidebar)
- `src/components/project/project-header.tsx` (project header)

## Goal
Extract a shared `useProjectActions` hook so all four locations use the same export, delete, archive, and active-operations logic. The batch page cards gain Export and Delete in their dropdowns.

## Decisions (Resolved)

### 1. Extraction Scope — ALL common handlers
Extract export, delete, archive toggle, and active-operations check into a single shared hook. Eliminates all duplication rather than only adding export/delete.

### 2. Consumer Scope — All four locations
The hook is consumed by:
- `project.tsx`
- `batch.tsx`
- `app-sidebar-projects.tsx`
- `project-header.tsx`

### 3. Archive UX — Hook manages delete state only
The hook returns delete-modal state (`isDeleteModalOpen`, `projectToDelete`, `promptDelete`, `handleConfirmDelete`) plus simple imperative handlers for export and archive. The project header keeps its own `ArchiveDialog` wrapper (it has a redirect-after-archive flow the list pages don't share).

### 4. Batch Card API — Add props to existing components
`SortableBatchCard` and `ArchivedBatchCard` receive new props:
- `onExport: (id: string) => void`
- `onDelete: (id: string) => void`

They continue receiving `onSelect` and `onToggleArchive` as before.

### 5. Delete Dialogue — Return state and handlers only
Each consumer renders its own `<DeleteDialogue>` using the hook's state. The hook does not return a rendered component.

## Implementation Plan

### Step 1: Create `src/hooks/project/use-project-actions.ts`

Create the shared hook with the following API:

```ts
export function useProjectActions() {
  // State
  const [projectToDelete, setProjectToDelete] = useState<string | null>(null)
  const [isDeleteModalOpen, setIsDeleteModalOpen] = useState(false)

  // Store selectors
  const deleteProject = useProjectStore(state => state.deleteProject)
  const updateProject = useProjectStore(state => state.updateProject)
  const projects = useProjectStore(state => state.projects)

  // Delete flow
  const promptDelete = (projectId: string) => {
    setProjectToDelete(projectId)
    setIsDeleteModalOpen(true)
  }

  const handleConfirmDelete = async () => {
    if (!projectToDelete) return
    await deleteProject(projectToDelete)
    setIsDeleteModalOpen(false)
    setProjectToDelete(null)
  }

  // Export flow
  const handleExport = async (projectId: string) => { ... }

  // Archive flow
  const handleArchive = async (projectId: string, archive: boolean) => {
    if (archive && checkActiveOperations(projectId)) {
      toast.error("Cannot archive — finish or cancel active operations first")
      return
    }
    const updated = await updateProject(projectId, { isArchived: archive })
    if (updated) {
      toast.success(archive ? "Project archived" : "Project unarchived")
    } else {
      toast.error(`Failed to ${archive ? "archive" : "unarchive"} project`)
    }
    return updated
  }

  // Active operations check
  const checkActiveOperations = (projectId: string) => {
    const project = projects.find(p => p.id === projectId)
    if (!project) return false
    return hasActiveOperations(project)
  }

  return {
    isDeleteModalOpen,
    setIsDeleteModalOpen,
    projectToDelete,
    promptDelete,
    handleConfirmDelete,
    handleExport,
    handleArchive,
    checkActiveOperations,
  }
}
```

### Step 2: Update `SortableBatchCard` and `ArchivedBatchCard`

Add `onExport` and `onDelete` props to both components. Wire the dropdown menu items:
- Export item: calls `onExport(project.id)`
- Delete item: calls `onDelete(project.id)`

Use `Upload` and `Trash` icons. Apply `text-destructive` to the delete item.

### Step 3: Update `batch.tsx`

- Replace inline `handleToggleArchive` and `checkActiveOperations` with the hook.
- Pass `handleExport`, `promptDelete`, and `handleToggleArchive` (from hook) into `SortableBatchCard` and `ArchivedBatchCard`.
- Render `<DeleteDialogue handleDelete={handleConfirmDelete} ... />` at the bottom.

- Update `SortableProjectCard` and `ArchivedProjectCard` (inline components) to use the hook's handlers from parent scope.

### Step 5: Update `app-sidebar-projects.tsx`

- Replace inline `handleExportProject`, `handleDeleteProject`, and delete-modal state with the hook.
- Wire `handleExport` and `promptDelete` into the dropdown menu items.
- Render `<DeleteDialogue handleDelete={handleConfirmDelete} ... />`.

- Replace `handleExportProject` and `handleDelete` with the hook's handlers.
- Keep `handleToggleArchive` for unarchive (direct call, no redirect).

- Render `<DeleteDialogue handleDelete={handleConfirmDelete} ... />`.
