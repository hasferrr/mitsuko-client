# Plan: Remove Manual Styling Overrides, Use shadcn/ui Defaults

## Problem
Components across the codebase use manual `padding`, `margin`, `width`, `height`, and hardcoded colors that override or duplicate styles already provided by shadcn/ui components. This creates inconsistent spacing, double-borders, redundant classes, and visual misalignment.

## Approach
Systematically clean up components by removing ad-hoc styling and relying on shadcn/ui's built-in variant/size system. Group fixes by category and tackle them in dependency order (shared components first, then consumers).

---

## Confirmed Decisions
- **Scope**: All 8 categories
- **Button `h-10`**: Use `size="lg"` (h-9) — no new xl variant. Accept slightly smaller buttons for design system consistency.
- **Title Input `h-12`**: Remove `h-12 text-xl font-semibold` and let them use default Input sizing to match adjacent buttons. The title styling (text-xl font-semibold) can be kept as a class but the height override should go — let the input's natural height match surrounding controls.
