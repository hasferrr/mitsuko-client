## Changes by Category

### Cat 2: Replace card-like divs with Card/CardContent

**`few-shot-input.tsx`** (line 214, 259, 262):

The `<div className={cn("p-3 border rounded-md space-y-3")}>` at line 214 is a content container with border, padding, and rounded corners — card-like. Convert to Card + CardContent.

```tsx
// Before (line 214)
<div className={cn("p-3 border rounded-md space-y-3")}>

// After
<Card size="sm">
  <CardContent className="space-y-3">
```

```tsx
// Before (line 262, closing div)
</div>

// After
  </CardContent>
</Card>
```

- Add import: `import { Card, CardContent } from "@/components/ui/card"`
- Also remove `pt-2` from `<p>` at line 259 (space-y-3 on CardContent handles spacing): `<p className="text-xs text-muted-foreground pt-2">` → `<p className="text-xs text-muted-foreground">`

### Cat 4: Button size variants instead of manual overrides

**`model-selection.tsx`** (lines 54-62):

```tsx
// Before
<Button
  variant="outline"
  size="default"
  className="w-10 flex items-center justify-center"
  onClick={() => setIsWhichModelsDialogOpen(true)}
  aria-label="Which Models Should I Use?"
>

// After — use size="icon" which provides size-8 + centering
<Button
  variant="outline"
  size="icon"
  onClick={() => setIsWhichModelsDialogOpen(true)}
  aria-label="Which Models Should I Use?"
>
```

This is an icon-only button (contains `<HelpCircle>` with `aria-label`). `size="icon"` gives `size-8` (square) with built-in centering, making `w-10 flex items-center justify-center` redundant.
